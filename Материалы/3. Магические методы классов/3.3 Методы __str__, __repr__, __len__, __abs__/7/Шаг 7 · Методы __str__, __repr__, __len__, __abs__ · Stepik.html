Подвиг 6. Объявите класс с именем Complex для представления и работы с комплексными числами. Объекты этого класса должны создаваться командой:

cm = Complex(real, img)

где real - действительная часть комплексного числа (целое или вещественное значение); img - мнимая часть комплексного числа (целое или вещественное значение).

Объявите в этом классе следующие объекты-свойства (property):

real - для записи и считывания действительного значения;
img - для записи и считывания мнимого значения.

При записи новых значений необходимо проверять тип передаваемых данных. Если тип не соответствует целому или вещественному числу, то генерировать исключение командой:

raise ValueError("Неверный тип данных.")

Также с объектами класса Complex должна поддерживаться функция:

res = abs(cm)

возвращающая модуль комплексного числа (вычисляется по формуле: sqrt(real*real + img*img) - корень квадратный от суммы квадратов действительной и мнимой частей комплексного числа).

Создайте объект cmp класса Complex для комплексного числа с real = 7 и img = 8. Затем, через объекты-свойства real и img измените эти значения на real = 3 и img = 4. Вычислите модуль полученного комплексного числа (сохраните результат в переменной c_abs).

P.S. На экран ничего выводить не нужно.

Сделал через свойства, решение засчитано. Решил повторить, но через дескрипторы, поскольку мы здесь маловато в этой теме тренировались. У меня вариант с дескриптором работает, а здесь - не проходит:

Failed.

Runtime error

Error: Traceback (most recent call last):

File "jailed_code", line 29, in <module>

assert type(Complex.real) == property and type(Complex.img) == property, "в классе Complex должны быть объявлены объекты-свойства real и img"

File "jailed_code", line 6, in __get__

return instance.__dict__[self.name]

AttributeError: 'NoneType' object has no attribute '__dict__'

 

То есть стоит жёсткая проверка на то, что это именно свойства, дескрипторы уже не подходят.

 

Сравнил у себя результаты вывода команды print(type(Complex)): в обоих случаях выдаётся 

<class 'type'>

Сравнил вывод команды print(type(Complex.real)), и вот тут получается, что для варианта со свойствами выдаётся 

<class 'property'>, а для дескрипторов как раз то, что выдала проверяющая система: 
AttributeError: type object 'type' has no attribute 'real'.

Получается, что при обращении к свойству через класс вместо объекта (Complex.real) происходит обращение к словарю самого класса и вытаскивание оттуда объекта property. А вот при аналогичном обращении к дескриптору выполняется обращение к методу __get__ дескриптора, но вместо аргумента instance передаётся None.

Поэтому добавил в метод __get__ две строки:

        if instance is None:

            return self

Так, по крайней мере, исчезла ругань про AttributeError. Ну а с тем, что это не свойства, уже ничего не поделаешь, эта проверка зашита в тесте.

Такие эксперименты помогают лучше понять то, что происходит "под капотом".
