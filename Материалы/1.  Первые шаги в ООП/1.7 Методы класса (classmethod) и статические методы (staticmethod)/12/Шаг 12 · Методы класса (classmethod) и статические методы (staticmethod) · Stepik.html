Подвиг 11 (на повторение). Объявите класс для мессенджера с именем Viber. В этом классе должны быть следующие методы:

add_message(msg) - добавление нового сообщения в список сообщений;
remove_message(msg) - удаление сообщения из списка;
set_like(msg) - поставить/убрать лайк для сообщения msg (если лайка нет то он ставится, если уже есть, то убирается);
show_last_message(число) - отображение последних сообщений;
total_messages() - возвращает общее число сообщений.

Эти методы предполагается использовать следующим образом (эти строчки в программе не писать):

msg = Message("Всем привет!")
Viber.add_message(msg)
Viber.add_message(Message("Это курс по Python ООП."))
Viber.add_message(Message("Что вы о нем думаете?"))
Viber.set_like(msg)
Viber.remove_message(msg)

Класс Message (необходимо также объявить) позволяет создавать объекты-сообщения со следующим набором локальных свойств:

text - текст сообщения (строка);
fl_like - поставлен или не поставлен лайк у сообщения (булево значение True - если лайк есть и False - в противном случае, изначально False);

P.S. Как хранить список сообщений, решите самостоятельно.

Судя по комментариям и вопросам у моих соучеников (и у меня тоже) сложилось не полное понимание ситуаций, когда надо использовать декораторы  @staticmethod и @classmethod, а когда можно (или следует) без них обойтись. В связи с этим хочу привести пример-сравнение. Возможно, в ваших видео подобное было, но я упустил – в этом случае прошу прощения. После рассмотрения задам вам один вопрос.

Создадим класс MyTst, единственным атрибутом которого является bufer = '==', а единственным методом, определенным в трех аналогичных примерах без декоратора, с декоратором @staticmethod и с декоратором @classmethod. Этот метод,  add_txt(а), состоит в добавлении к строке bufer строки а:

class MyTst:                                      |        class MyTst:                                              |        class MyTst:                              
    bufer = '=='                                     |            bufer = '=='                                             |            bufer = '=='                             
    # Без декоратора                       |            @staticmethod                                     |            @classmethod                     
    def add_txt(self, a):                      |            def add_txt(a):                                       |            def add_txt(cls, a):
        self.bufer = self.bufer + a        |                MyTst.bufer = MyTst.bufer + a       |                cls.bufer = cls.bufer + a

Небольшие синтаксические отличия в примерах обусловлены отсутствием / наличием декоратора. В основном теле программ (есть синтаксические отличия по той же причине) создаются по 2 экземпляра класса s1, s2, методом add_txt добавляем к их атрибуту bufer, соответственно, 'aaaa' и 'bbbb', а к атрибуту bufer самого класса 'AAAA'. Выводим значения s1.bufer, s2.bufer, MyTst.bufer и их id:

s1 = MyTst()
s2 = MyTst()
s1.add_txt('aaaa')
s2.add_txt('bbbb')
MyTst.add_txt(MyTst,'AAAA')
print(s1.bufer, s2.bufer, MyTst.bufer)
print(id(s1.bufer), id(s2.bufer), id(MyTst.bufer))        #( две аналогичных программы опускаю).

==========================================================================================================На консоли:

без декоратора:           ==aaaa   ==bbbb   ==AAAA
                                         2719885206192   2719885205872   2719885207088

@staticmethod:             ==aaaabbbbAAAA  ==aaaabbbbAAAA  ==aaaabbbbAAAA
                                         2719885058224      2719885058224     2719885058224

@classmethod:             ==aaaabbbbAAAA  ==aaaabbbbAAAA  ==aaaabbbbAAAA
                                         2719885132208      2719885132208     2719885132208

========================================================================================================= Из приведенного примера видно, что в отсутствии декоратора атрибуты bufer экземпляров класса и самого класса имеют разные  id и, соответственно, изменяются независимо при применении к экземплярам класса и самому классу метода  add_txt. Использование декораторов приводит к тому, что атрибуты bufer экземпляров класса ссылаются на атрибут самого класса.
