Хочу поправить фразу из видео: "после вызова магического метода __del__ объект ГАРАНТИРОВАННО перестанет существовать". После вызова __del__ объект не перестанет существовать. Объект перестанет существовать тогда, когда его удалит сборщик мусора => когда на объект не будет ссылок.

Это важно, поскольку если у нас есть две ссылки на объект и мы удаляем одну из ссылок (например, через del), то объект всё ещё остаётся жив. Аналогично и с вызовом obj.__del__() - мы зайдём в метод, если его прописали, но удаление не произойдёт и объект останется существовать.

class Test:
    def __init__(self):
        self.name = self

    def __del__(self):
        print(f'Зашли в удаление объекта {self}')
        del self


test = Test()
test2 = test
print('До удаления')
test.__del__()
del test
print('После удаления')
print(f'Имя: {test2.name}')
