Подвиг 9. Вам необходимо реализовать односвязный список (не список языка Python, объекты в списке не хранить, а формировать связанную структуру, показанную на рисунке) из объектов класса ListObject:

Для этого объявите в программе класс ListObject, объекты которого создаются командой:

obj = ListObject(data)

Каждый объект класса ListObject должен содержать локальные свойства:

next_obj - ссылка на следующий присоединенный объект (если следующего объекта нет, то next_obj = None);
data - данные объекта в виде строки.

В самом классе ListObject должен быть объявлен метод:

link(self, obj) - для присоединения объекта obj такого же класса к текущему объекту self (то есть, атрибут next_obj объекта self должен ссылаться на obj).

Прочитайте список строк из входного потока командой:

lst_in = list(map(str.strip, sys.stdin.readlines()))

Затем сформируйте односвязный список, в объектах которых (в атрибуте data) хранятся строки из списка lst_in (первая строка в первом объекте, вторая - во втором и  т.д.). На первый добавленный объект класса ListObject должна ссылаться переменная head_obj.

P.S. В программе что-либо выводить на экран не нужно.

@Max_Avern, а что вы не понимаете конкретно? вроде как тут особо нечего понимать, если вы запомнили всё из что для чего и как

У вас есть класс, например Car, и вам нужно сделать два экземпляра. Один экземпляр будет с именем "BMW X6", другой "Лада Калина".

Как нам получить такое поведение?

На данный момент у нас есть только класс Car



class Car:
    pass



Чтобы эти наименования принадлежали экземплярам нам нужно их им присвоить, например, в атрибут name. В лекции говорилось, что это можно сделать после создания экземпляров



car1 = Car() # создали экземпляр
car1.name = "BMW X6" # создали в нём атрибут и положили в него данные
car2 = Car()
car2.name = "Лада Калина"



Т.е. в экземпляре мы создаём атрибут name и кладём в него что хотим по нашему замыслу (логика схожа с логиков работы со словарями при добавлении ключей)



Однако если мы хотим каждому экземпляру дать некоторый набор атрибутов с какими-то значениями для каждого из экземпляров, то согласитесь, будет несколько громоздко. А если у нас будет больше 2х экземпляров? Код будет довольно большим

Именно поэтому добавили спец ф-ию инициализатор __init__(), один из магических методов, которая работает точно по такому же сценарию: 1) создали экзмепляр, 2) добавили ему набор атрибутов.



Хорошо и как это работает, ведь у нас разные данные для одинаковых атрибутов для разных экземпляров? Как применить этот спец метод?

В лекции сообщалось, что этот метод вызывается для каждого созданного экземпляра. Как метод понимает для какого именно экземпляра он вызван? Ему это сообщает сам интерпретатор Python через параметр self. Т.е. по факту, что мы имеем



class Car:
    def __init__(self):
        # self - ссылка на текущий созданных экземпляр
        pass



car2 = Car() # тут создался экзмепляр и скрыто от нас тут же, не переходя к исполнению следующей строки кода, вызвался метод.



Ок. Вроде всё понятно? Мне кажется да.

Хорошо и как же теперь дать car2 набор атрибутов, которые мы хотим, с теми значения которые мы хоти для этого конкретного экземпляра?

Так как мы уже поняли, что запись Car() не только создаёт экземпляр, но и по факту является вызовом этого метода, куда сразу же попадает ссылка на car2, мы можем передать и необходимые данные для набора атрибутов, которые мы хотим видеть в каждом из экземпляров в виде параметров ф-ии (ведь метод - это просто ф-ия, на что обратил внимание автор курса). Т.к. self всегда передаётся скрыто от нас в первом параметре, а все последующие переданные уже нами данные идут со второго параметра, не смотря на то, что в строке создания экземпляра они идут как будто с первого параметра, мы получим следующее



class Car:
    def __init__(self, name):
        self.name = name # тоже самое, что, например, car2.name = какие-то_данные_из_name



Какие же это данные такие из name? Те, что мы передали при создании экземпляра



car2 = Car("BMW X6")
