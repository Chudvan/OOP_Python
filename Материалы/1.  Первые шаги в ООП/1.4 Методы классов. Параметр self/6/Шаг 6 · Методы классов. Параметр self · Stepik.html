Подвиг 5. Объявите класс с именем Graph и методами:

set_data(data) - передача набора данных data для последующего отображения (data - список числовых данных);
draw() - отображение данных (в том же порядке, что и в списке data)

и атрибутом:

LIMIT_Y = [0, 10]

Метод set_data() должен формировать локальное свойство data объекта класса Graph. Атрибут data должен ссылаться на переданный в метод список. Метод draw() должен выводить на экран список в виде строки из чисел, разделенных пробелами и принадлежащие заданному диапазону атрибута LIMIT_Y (границы включаются).

Создайте объект graph_1 класса Graph, вызовите для него метод set_data() и передайте список:

[10, -5, 100, 20, 0, 80, 45, 2, 5, 7]

Затем, вызовите метод draw() через объект graph_1. На экране должна появиться строка с соответствующим набором чисел, записанных через пробел. Например (вывод без кавычек):

"10 0 2 5 7"

Возможно я был невнимателен, но по-моему в лекции очень не хватает обсуждения того, как обращаться к статическим свойствам класса из его же методов. Просто написать LIMIT_Y[0] или LIMIT_Y[1] в методе нельзя. Есть следующие опции:

  • Писать self.LIMIT_Y[0] или self.LIMIT_Y[1] - это работает, но потенциально опасно, т.к. если один из объектов заведёт у себя внутри LIMIT_Y, метод класса сломается.
  • Альтернативно, можно писать Graph.LIMIT_Y или type(self).LIMIT_Y. Первый вариант внешне понятен, хотя не совсем явно почему методы не умеют подразумевать свойства класса по умолчанию. Второй вариант, пишут, позволяет проще переименовывать классы, но я не знаю, нет ли в этом варианте каких-то подводных камней.
  • Кроме того, в лекции обсуждали функции setattr() и getattr(); но для их использования нам придётся снова писать первым аргументом self, или Graph, или type(self), т.е. вопрос о рекомендуемом подходе это не отменяет.

class Graph:
    LIMIT_Y = [0, 10]

    def set_data(self, data):
        self.data = data

    def draw(self):
        self.new_str = ""
        for number in self.data:
            if number <= self.LIMIT_Y[1] and number >= self.LIMIT_Y[0]:
                self.new_str += f"{number} "
        return self.new_str


graph_1 = Graph()
