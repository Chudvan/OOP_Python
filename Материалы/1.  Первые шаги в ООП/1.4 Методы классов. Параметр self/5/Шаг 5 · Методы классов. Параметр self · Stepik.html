Может кому пригодится моё толкование.

1. Создаём класс с именем MediaPlayer

2. Добавляем в наш класс метод(функцию) с названием open принимающий в качестве аргумента переменную file. В этой переменной кагбэ имя файла, и мы при вызове этого метода должны сохранить его в атрибут(переменную) экземпляра класса.

На данном этапе мы сталкиваемся с не вполне очевидным поведением: Во первых, хотим мы того, или нет, а первым аргументом в нашем методе должен идти этот их self. В противном случае - экземпляр класса просто не сможет вызвать этот метод. Во вторых, чтобы создать или изменить атрибут(переменную) экземпляра класса нельзя просто взять, и написать имя переменной из метода(функции). Вместо x = 123 нам из функции(будущего метода) надо написать self.x = 123. Это создаст в пространстве имён текущего экземпляра класса переменную x со значением 123.

Короче говоря, метод(функция) open должен создавать/изменять внутри экземпляра класса атрибут(переменную) с именем filename и со значением переданным методу open в параметре file.

Но на этом веселье не заканчивается.

3. Добавляем ещё один метод с названием play. Это будет уже кагбэ воспроизведение, а на самом деле вывод информации через print. Никаких параметров кроме self этот метод уже не принимает, зато выводит на экран необходимые буквы, а именно "Воспроизведение <название медиа-файла>", где вместо <название медиа-файла> конечно должно стоять значение атрибута filename, которое создаёт и изменяет ранее созданный метод open.

Теперь, чтобы доказать всем как великолепно это (не) работает, начинаем эксплуатировать наш классный класс.

4. Создаём два экземпляра класса MediaPlayer с именами: media1 и media2. Из каждого вызываем метод open() с аргументом "filemedia1" для объекта media1 и "filemedia2" для объекта media2. Если мы всё сделали правильно, то для каждого экземпляра класса будет создано своё значение, и записано в правильный атрибут(переменную).

5. Теперь для каждого созданного экземпляра класса MediaPlayer(media1 и media2) вызываем метод play(). Ничего конечно не получится, но если бы мы всё сделали правильно, то на экране напечатались бы две строки:

Воспроизведение filemedia1
Воспроизведение filemedia2

И всего делов :)
