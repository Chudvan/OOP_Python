Возможно, глупый вопрос, но уже во второй задаче подряд столкнулся с такой проблемой:

если в дескрипторе в методе __set_name__ не менять имя локального свойства, например, на приватное ('_' + name), а оставить его как есть, то при использовании в методах __set__ и __get__ функций setattr и getattr соответственно, программа на месте срабатывания этих функций уходит в рекурсию, при этом при помощи instance.__dict__[self.name] = value всё работает.

Ну и при изменении имени локального атрибута на "_name" или "__name" тоже всё работает.

Буду благодарен, если кто-нибудь разъяснит этот тонкий момент, вроде никакого конфликта имен быть не должно, я же применяю функцию setattr к экземпляру instance, а self.name  - это его (instance) локальный атрибут, а не атрибут экземпляра класса дескриптора

ТАК НЕ РАБОТАЕТ:

def __set_name__(self, owner, name):
    self.name = name

def __set__(self, instance, value):
    if self.validator.validate(value):
        setattr(instance, self.name, value)

 

А ТАК НОРМ:

def __set_name__(self, owner, name):
    self.name = '_' + name

def __set__(self, instance, value):
    if self.validator.validate(value):
        setattr(instance, self.name, value)

 

И ТАК НОРМ:

def __set_name__(self, owner, name):
    self.name = name

def __set__(self, instance, value):
    if self.validator.validate(value):
        instance.__dict__[self.name] = value
