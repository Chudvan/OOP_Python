Возник вопрос пусть нужно валидировать данные при записи в объект, то как эту валидацию вызвать при инициализации. Например мы хотим, чтобы имя было полностью заглавными буквами. Когда мы пишем p.name валидация срабатывает. Но вот при создании валидация не вызовется. Как правильно это делать?

class Person:

    def __init__(self, name):
        self.__name = name

    @property
    def name(self):
        return self.__name

    @name.setter
    def name(self, value):
        print('validation')

        self.__name = value.upper()


p = Person("Ivan")
print(p.name)
p.name = "Nazar"
print(p.name)

А как присваивать значение через сеттер, если локальных свойств больше одного?

например:

class Person:

    def __init__(self, name, g, h):
        self.__name = name
        self.__g = g
        self.__h = h


@property
def name(self):
    return self.__name


@name.setter
def name(self, value, g, h):
    self.__name = value
    self.__g = g
    self.__h = h

@Anonymous_446803014, да он где то говорил для чего так делается. Но это было в уроках ранее по сеттерам и геттерам.

Ответ простой, тут пример примитивен. Иногда нужно спрятать логику определенную, особенно это касается сеттеров.  Ты присваиваешь по имени атрибута какое-то значение, а внутри выполняется логика, проверки и т.д. Актуально когда пишешь отдельные модули .py или некие API проектируешь сам.  В теории все можно оставить работать через set_ метод, имеется ввиду проверки и доп логика, просто через имя атрибута удобнее.
